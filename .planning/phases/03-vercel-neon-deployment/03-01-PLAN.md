```
"buildCommand": "cd apps/dashboard && bun run build"
```

to:

```
"buildCommand": "cd packages/db && bunx drizzle-kit push --force --url=$DATABASE_URL_UNPOOLED && cd ../.. && turbo build --filter=@sessionforge/dashboard"
```

Per D-02: Build must run drizzle-kit push before turbo build.
Per D-03: Uses DATABASE_URL_UNPOOLED (direct/unpooled Neon connection) for migrations -- pooled connections break DDL statements through PgBouncer.

Key flags:
- `--force`: Skips interactive confirmation prompts (without this, build hangs indefinitely in CI -- documented in CLAUDE.md)
- `--url=$DATABASE_URL_UNPOOLED`: Overrides drizzle.config.ts DATABASE_URL with the unpooled connection. The Neon-Vercel integration auto-provides this env var.
- `turbo build --filter=@sessionforge/dashboard`: Builds only the dashboard app via Turbo pipeline (proper monorepo build resolving workspace deps)

Do NOT change any other fields in vercel.json (framework, installCommand, outputDirectory, functions, headers, regions, crons).
