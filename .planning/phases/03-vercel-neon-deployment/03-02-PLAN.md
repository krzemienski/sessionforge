The user must provision a Neon Postgres database. Claude cannot do this -- it requires account creation and dashboard interaction.

**Steps for the user:**

1. Go to https://console.neon.tech and sign in (or create an account)
2. Create a new project:
   - Name: `sessionforge` (or preferred name)
   - Region: `us-east-1` (matches Vercel region `iad1` in vercel.json)
   - Postgres version: 17 (latest stable)
3. From the Connection Details panel, copy TWO connection strings:
   - **Pooled** (for runtime): The default connection string containing `-pooler` in the hostname
     Example: `postgresql://neondb_owner:PASSWORD@ep-xxx-yyy-pooler.us-east-1.aws.neon.tech/neondb?sslmode=require`
   - **Direct/Unpooled** (for migrations): Toggle "Direct connection" or find the string WITHOUT `-pooler`
     Example: `postgresql://neondb_owner:PASSWORD@ep-xxx-yyy.us-east-1.aws.neon.tech/neondb?sslmode=require`
4. (Optional, per DEPL-02) Disable scale-to-zero:
   - Go to Project Settings -> Compute
   - Set suspend timeout to "Never" (may require paid plan; if on free plan, skip this)

**Provide both connection strings when done.** They will be configured as Vercel env vars in the next step.
  </action>

    - Region is us-east-1 (matching Vercel iad1)

The user must create/link the Vercel project and configure env vars. This can be done via Vercel CLI or dashboard.

**Option A: Vercel CLI** (if preferred)

```bash
# From repo root
vercel link
# Follow prompts: select team/scope, link to existing or create new project

# Add required env vars (production scope)
vercel env add DATABASE_URL production
# Paste: the POOLED Neon connection string from Task 1

vercel env add DATABASE_URL_UNPOOLED production
# Paste: the DIRECT/UNPOOLED Neon connection string from Task 1

vercel env add BETTER_AUTH_SECRET production
# Paste: output of `openssl rand -base64 32`

vercel env add BETTER_AUTH_URL production
# Paste: https://sessionforge.vercel.app (or your custom domain)

vercel env add NEXT_PUBLIC_APP_URL production
# Paste: same URL as BETTER_AUTH_URL

vercel env add DISABLE_AI_AGENTS production
# Paste: true
```

**Option B: Vercel Dashboard**

1. Go to https://vercel.com/dashboard
2. Import the sessionforge repository (or link existing project)
3. Go to Project -> Settings -> Environment Variables
4. Add these REQUIRED vars (Production scope):

| Variable | Value | Notes |
|----------|-------|-------|
| `DATABASE_URL` | Pooled Neon string (with -pooler) | Per D-03: runtime queries |
| `DATABASE_URL_UNPOOLED` | Direct Neon string (no -pooler) | Per D-03: migrations in build |
| `BETTER_AUTH_SECRET` | `openssl rand -base64 32` output | Auth token signing |
| `BETTER_AUTH_URL` | `https://<your-vercel-url>` | Per D-07: server-side auth |
| `NEXT_PUBLIC_APP_URL` | Same as BETTER_AUTH_URL | Per pitfall 3: baked into build |
| `DISABLE_AI_AGENTS` | `true` | Claude CLI not available on Vercel |

5. (Optional) Add these for enhanced features:

| Variable | Value | Notes |
|----------|-------|-------|
| `CRON_SECRET` | `openssl rand -hex 32` output | Protects /api/cron/automation |
| `UPSTASH_REDIS_URL` | Upstash endpoint | Cache (app works without it) |
| `UPSTASH_REDIS_TOKEN` | Upstash token | Cache token |

Per D-05: All required env vars from .env.example must be configured.
Per D-06: Do NOT set DATABASE_DRIVER -- db-adapter.ts auto-detects Neon URLs.
CRITICAL: Do NOT set ANTHROPIC_API_KEY -- the Agent SDK uses CLI-inherited auth, which is disabled on Vercel via DISABLE_AI_AGENTS=true.
